# ラボ 01 - Microsoft Entra ID ID を管理する

## ラボ シナリオ

このラボのタスクは以下の通りです。

+ タスク １：Azure Portalへサインイン
+ タスク ４：ユーザー作成とロール付与
+ タスク ５：ユーザー作成とライセンス付与
+ タスク ６：セキュリティグループ（動的ユーザー）の作成



## 予想時間: 30 分



## アーキテクチャ
![image](./media/lab01.png)



## 手順

#### タスク 1：Azure Portalへサインイン

1. [Azure portal](https://portal.azure.com) に講師から通知されたアカウントでサインインします。

2. Azure portal の画面上部にある検索ボックスに「**Microsoft Entra ID**」 と入力します。

   > 注：サインイン後、「**ツアーへようこそ**」と表示された場合は「**後で行う**」をクリックします。

3. 一覧に「**Microsoft Entra ID**」と表示されます。検索結果をクリックします。

#### タスク 2：Premium ライセンスを確認

> **シナリオ：このタスクでは、テナント上の割り当て可能なライセンスを確認します。**

1. **Microsoft Entra IDの概要**ブレードの左側ツリーから「**ライセンス**」をクリックします。
1. 「**ライセンス | 概要**」ブレードの左ツリーにある「管理 - すべての製品」をクリックします。
1. 「**ライセンス | すべての製品**」 ブレード内の一覧に「**Microsoft Entra ID P2**」が表示されていることを確認します。

1. Azure portal の画面上部にある検索ボックス「**Microsoft Entra ID**」 と入力します。
1. 一覧に「**Microsoft Entra ID**」と表示されます。検索結果をクリックします。
1. **Microsoft Entra IDの概要**ブレードが表示されます。
1. 画面中央にあるライセンスが「**Microsoft Entra ID P2**」であることを確認します。


#### タスク 3：ユーザー作成とロール付与

> **シナリオ：このタスクでは、First AADテナントに「az104-01a-aaduser1」という新たなユーザーを作成します。**
>
> 　　　　　**さらに作成したユーザーに「ユーザー管理者」ロールを付与し、特権アカウントにします。**

1. **FMicrosoft Entra IDの概要**ブレードの左側ツリーから「**ユーザー**」をクリックします。

1. 「**ユーザー**」 ブレードにある「**＋新しいユーザー**」をクリックし、さらに「**新しいユーザーの作成**」をクリックします。

1. 「**新しいユーザー**」 ブレードの「**基本**」タブで以下の項目を入力し、画面下部にある「**次：プロパティ**」をクリックします。

   > 注：表に無い項目は[デフォルトの値]または[未入力]となります。
   >
   > 各名称の"XX"にはアカウント名と同じ数字を入力してください

   | 設定                                | 値                                                         |
   | ----------------------------------- | ---------------------------------------------------------- |
   | ユーザー プリンシパル名             | **az104-01a-aaduserXX-1**                                  |
   | メール ニックネーム                 | 自動入力のまま                                             |
   | ユーザー プリンシパル名から引き継ぐ | ✔を入れる（入れたまま）                                    |
   | 表示名                              | **az104-01a-aaduserXX-1**                                  |
   | パスワード                          | **Pa55w.rd12345**                                          |
   | パスワードの自動生成                | ✔を**外す**<br />※外すとパスワードの入力が可能となります。 |
   | 有効なアカウント                    | ✔を入れる（入れたまま）                                    |

   

1. 「**新しいユーザー**」 ブレードの「**プロパティ**」タブで以下の項目を入力し、画面下部にある「**レビューと作成**」をクリックします。

   > 注：表に無い項目は[デフォルトの値]または[未入力]となります。
   >
   > 各名称の"XX"にはアカウント名と同じ数字を入力してください

   | 設定     | 値                 |
   | -------- | ------------------ |
   | 名       | **az104-01a**      |
   | 姓       | **aaduserXX-1**    |
   | 役職     | **クラウド管理者** |
   | 利用場所 | **日本**           |

   

1. 「**新しいユーザー**」 ブレードの「**確認と作成**」タブで設定した項目を確認し、画面下部にある「**作成**」をクリックします。

1. 画面上部の通知アイコン より「**ユーザーが正常に作成されました**」と表示されるのを待ちます。

1. 「**ユーザー**」 ブレードの一覧に作成したユーザー「**az104-01a-aaduserXX-1**」をクリックします。

1. 「**az104-01a-aaduserXX-1**」 ブレードの一覧にある「**ユーザー プリンシパル名**」をコピーしてメモ帳などに保存します。

   > 注：ユーザー プリンシパル名は"ユーザー名"と"テナント名(ドメイン名)"で構成されるIDです。

1. 「**az104-01a-aaduserXX-1**」 ブレードの左ツリーにある「**割り当てられたロール**」をクリックします。

1. 「**az104-01a-aaduserXX-1 | 割り当てられたロール**」 ブレードにある「**＋割り当ての追加**」をクリックします。

1. 画面右に表示される「**ディレクトリ ロール**」より「**ユーザー管理者**」を選択し、「**追加**」をクリックします。

1. 「**az104-01a-aaduserXX-1 | 割り当てられたロール**」ブレードに遷移します。画面内の更新またはWebブラウザの再読み込みを行います。

1. 一覧に「**ユーザー管理者**」と表示されたことを確認します。

1. Azure portal の画面上部にある検索ボックス「**Microsoft Entra ID**」 と入力します。

1. 一覧に「**Microsoft Entra ID**」と表示されます。検索結果をクリックします。



#### タスク ５：ユーザー作成とライセンス付与

>**シナリオ：このタスクでは、「ユーザー管理者」ロールを持つ「az104-01a-aaduserXX-1」でサインインをします。**
>
>　　　　　**サインイン後、新たに「az104-01a-aaduserXX-2」を作成します。**

1. Webブラウザを「**InPrivate**」または「**シークレット**」モードで用意します。

     > 注：演習の中ではWebブラウザの機能を使って別セッションでサインインします。
     >
     > 　　どのWebブラウザもウィンドウ右上の設定ボタンから表示することが可能です。
     >
     > 　  　-Microsoft Edgeでは「InPrivate」ウィンドウ
     >
     > 　  　-Google Chromeでは「シークレット」ウィンドウ
     >
     > 　  　-Mozilla Fire Foxでは「プライベート」ウィンドウ

1. 用意したWebブラウザで作成したアカウント(`az104-01a-aaduserXX-1@XXXXXXXX.onmicrosoft.com`)で [Azure portal](https://portal.azure.com) にログインします。

      > 注：サインイン時にパスワードの変更を求められる場合があります。
      >
      > 　　「Pa55w.rd12345」から「Pa55w.rd1234」などの有効な文字列に変更してください。

1. Azure portal の画面上部にある検索ボックス「**Microsoft Entra ID**」 と入力します。

1. 一覧に「**Microsoft Entra ID**」と表示されます。検索結果をクリックします。

1. **Microsoft Entra IDの概要** ブレードの左側ツリーから「**ユーザー**」をクリックします。

1. 「**ユーザー**」 ブレードにある「**＋新しいユーザー**」をクリックし、さらに「**新しいユーザーの作成**」をクリックします。

1. 「**新しいユーザー**」 ブレードの「**基本**」タブで以下の項目を入力し、画面下部にある「**次：プロパティ**」をクリックします。

      > 注：表に無い項目は[デフォルトの値]または[未入力]となります。

      | 設定                                | 値                                                         |
      | ----------------------------------- | ---------------------------------------------------------- |
      | ユーザー プリンシパル名             | **az104-01a-aaduserXX-2**                                  |
      | メール ニックネーム                 | 自動入力のまま                                             |
      | ユーザー プリンシパル名から引き継ぐ | ✔を入れる（入れたまま）                                    |
      | 表示名                              | **az104-01a-aaduserXX-2**                                  |
      | パスワード                          | **Pa55w.rd12345**                                          |
      | パスワードの自動生成                | ✔を**外す**<br />※外すとパスワードの入力が可能となります。 |
      | 有効なアカウント                    | ✔を入れる（入れたまま）                                    |

      

1. 「**新しいユーザー**」 ブレードの「**プロパティ**」タブで以下の項目を入力し、画面下部にある「**レビューと作成**」をクリックします。

      > 注：表に無い項目は[デフォルトの値]または[未入力]となります。

      | 設定     | 値                 |
      | -------- | ------------------ |
      | 名       | **az104-01a**      |
      | 姓       | **aaduserXX-2**    |
      | 役職     | **システム管理者** |
      | 利用場所 | **日本**           |

      

1. 「**新しいユーザー**」 ブレードの「**確認と作成**」タブで設定した項目を確認し、画面下部にある「**作成**」をクリックします。

1. 画面上部の通知アイコン より「**ユーザーが正常に作成されました**」と表示されるのを待ちます。

1. 「**ユーザー**」 ブレードの一覧に作成したユーザー「**az104-01a-aaduserXX-2**」をクリックします。

1. 「**az104-01a-aaduserXX-2**」 ブレードの左ツリーにある「**ライセンス**」をクリックします。

1. 「**az104-01a-aaduser2 | ライセンス**」 ブレードにある[**+割り当て**]をクリックします。

1. 「**ライセンス割り当ての更新**」 ブレードで 「**Microsoft Entra ID P2**」 のチェックを入れ、「**保存**」をクリックします。

1. 「**X**」(閉じる) をクリックして「**ライセンス割り当ての更新**」 ブレードを閉じ、「**az104-01a-aaduserXX-2 | ライセンス**」ブレードに戻ります。

1. 「**az104-01a-aaduserXX-2 | ライセンス**」 ブレードの一覧に割り当てたライセンスが表示されたことを確認します。

      >注：ライセンスの反映から表示されるまで時間がかかる場合があります。
      >
      >　　Webブラウザを何度か再読み込みする必要があります。

1. 画面右上のユーザー名をクリックし、「**サインアウト**」を行います。

1. 「**InPrivate**」または「**シークレット**」モードのWebブラウザを閉じます。

#### タスク ６：セキュリティグループ（動的ユーザー）の作成

>**シナリオ：このタスクではセキュリティグループ(動的ユーザー)を作成します。**

> 注：ここからは「LabUser-XXXXXXXX」のユーザー、Webブラウザに戻り操作を行います。

1. **Microsoft Entra IDの概要** ブレードの左側ツリーから「**グループ**」をクリックします。

1. 「**グループ | すべてのグループ**」ブレードにある「**新しいグループ**」をクリックします。

1. 「**新しいグループ**」 ブレードで以下の項目を入力します。

   > 注：表に無い項目は[デフォルトの値]または[未入力]となります。
   >
   > 注：メンバーシップの種類を「**動的ユーザー**」に指定する場合、**Premium P1/P2**のライセンスが必要となります。
   >
   > 　　ライセンスがない場合、グレーアウトされ「割り当て済み」の選択となります。

   | 設定                                               | 値                                                          |
   | -------------------------------------------------- | ----------------------------------------------------------- |
   | グループの種類                                     | **セキュリティ**                                            |
   | グループ名                                         | **IT クラウド管理者XX**                                     |
   | グループの説明                                     | **Contoso IT クラウド管理者**                               |
   | グループに Azure AD ロールを割り当てることができる | **いいえ**                                                  |
   | メンバーシップの種類                               | **動的ユーザー**                                            |
   | 所有者                                             | **所有者が選択されていません**<br />※所有者は指定しません。 |

   

1. 「**動的クエリの追加**」をクリックし、以下の項目でルールの構成を設定します。

   | 設定        | 値                      |
   | ----------- | ----------------------- |
   | および/また | **空欄**（グレーアウト) |
   | プロパティ  | **jobTitle**            |
   | 演算子      | **Equals**              |
   | 値          | **クラウド管理者**      |

   

1. 「**ルールの検証**」タブをクリックし、画面内にある「**+ユーザーの追加**」をクリックします。

1. 画面右側に表示された「**ユーザーを追加**」で以下のユーザーを選び、「**選択**」をクリックします。

   | 設定             | 値                                                       |
   | ---------------- | -------------------------------------------------------- |
   | 選択するユーザー | **az104-01a-aaduserXX-1**<br />**az104-01a-aaduserXX-2** |

   

1. 「**動的メンバーシップ ルール**」ブレードの画面中央にある「状態」を確認します。

   > 注：ここではadduserXX-1は✔、adduserXX-2は✖になります。adduser1のjob Title(役職)はクラウド管理者のためです。

1. 「**動的メンバーシップ ルール**」ブレードの左上にある「保存」をクリックします。

1. 「**新しいグループ**」ブレードで「**作成**」をクリックします。

1. 「**グループ | すべてのグループ**」ブレードに遷移し、一覧に作成したグループ「**IT クラウド管理者**」が表示されたことを確認します。

   > 注：表示されない場合、画面内の「更新」またはWebブラウザの再読み込みを行います。
